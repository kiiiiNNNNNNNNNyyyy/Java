Object Oritented Programming

 Object Oritented Programming Paradigms

  Abstraction

  Removing the irrelevant and keeping only the essentials. A powerful way to manage abstraction is trhough the use of hierarchial classifications.

  Encapsulation

  Encapsulation is the mechanism that binds together the code and the data it manipulates, and keeps both safe from outside interference and misuse. Access to the code and the data inside the wrapper is tightly controlled through a well defined interface. When you create a class, you will specify the code and the data that constitutes the class. Collectively, these elements are called the members of the class.

  Inheritance

  Inheritance is the process by which one object acquires the properties of another object. It supports the concept of hierarhical classification. Inheritance interacts with encapsulation as well. If a given class encapsulates some attributes, then any subclass will have the same attributes plus any that it adds as part of its specialization. Aa new subclass inherits all the attribute of its ancestor.

  Polymorphism

  Polymorphism is a feature that allows one interface to be used for a general class of actions. In simpler words' "Some features, different behaviors." Therefore, polymorphism is the property of the object to take on many forms.

Classes

 Class

  Fundamentals

  A class is a template for an object, and an object is an instance of a class. Because an object is an instance of a class, object and instance are alomst same. Data or variables defied inside the class are called instance variables.

  The this keyword

  'this' can be used inside any method to refer to the current object. 'this' is always the reference to the object on which the method is invoked. You can use 'this' anywhere a reference to an object of the current class is permmitted.

  Garbage Collection

  Objects are dynamically created in java by using the 'new' operator. Java handles deallocation autommmatically. Where no references to an object exists, that object is assumed to be no longer needed, and the memory occupied by the pobject can be used. There is no explicit need to destroy the objects as in C++.

  Finalize() method

  Sometimes an object will no need to perform some action when it is destroyed. For eg, if an object is holding some non-java resources such as a file handle or character font, then you might want to make sure these resources are freed before an object is destroyed. To handle such situations, java provides a mechanism called finalization. By using finalization, you can define specific actions that will occur when the object's memory is just about to be reclaimed by the garbage collector.

 String Class

  Fundamentals

  The most important thing to remember about strings in java is that strings are immutable. Once a String object is created, its content cannot be altered. To alter Strings, Java defines peer classes of String, called StringBuffer and StringBuilder, which allows strings to be altered. Both these classes are available in java.lang package. All three (String, StringBuffer and StringBuilder) implements the CharSequence interface.

 charAt() - To extract a single character from a string, you can refer directly to an individual characters via charAt() method.
 getChars() - If you need to extract more than one character at a time, you can use getChars().
 getBytes() - This is an alternative to getChars() that stores the characters in an array of bytes. It uses the default character-to-byte conversions provided by the platform.
 regionMatches() - The regionMatches() method comparesa specific region inside a string with another specific region in another string. It returns true or false.
 startsWith() - The startsWith() method determines whether a given String begins with a specified string.
 endsWith() - The endsWith() determines whether the string given ends with a specified string.
  equals() Versus == - The equals method compares the characters inside a String object. The == operator compates two object references to see whether they refer to the same instance.
  indexOf() - The indexOf() function searches for the first occurrence of a character or a substring.
  lastIndexOf() - The indexOf() function searches for the last occurrence of a character or a substring.
  substring() - This functions returns a copy of the substring that begins at startIndex and runs to the end of the invoking string or the mentioned the lastIndex.
 trim() - The trim method returns a copy of the invoking string from which any leading and trailing whitespace has been removed.
  join() - This method is used to concatenate two or more strings, seprating each string with a delimiter, such as a comma or a colon, depending on input.
  Some other useful functions - isEmpty(), replaceFirst(), replaceAll(), split().

  StringBuffer

  StringBuffer supports modifiable strings. StringBuffer represents growable and writable character sequences. StringBuffer may have characters and substrings inserted in the middle or appended to the end.
  The current length of a StringBuffer can be found via the length() method, while the total allocated capacity can be found through the capacity() method.
  ensureCapacity() - This method can set the size of the buffer. This is useful if you know in advane that you will be appending a large number of small strings to StringBuffer
  setLength() - To set the length of the string within a StringBuffer ibject, we have to use setLength()
  setChatAt() - StringBuffer allows to set the character at a particular point which we can get using chatAt() or getChars()
  append() - This method concatenates the string representation of any other type of data to the end of the invoking StringBuffer object.
  insert() - This method inserts one string into another. It is overloaded to accept values of all the primitive types, plus Strings, Objects, and CharSequence.
  reverse() - This method returns the reverse the object on which it is called
  delete() - This method deletes a sequence of characters from the invoking object. We can also delte single character at a prticular index by using deleteCharAt()
  replace() - You can replace one set of characters with another set inside a StringBuffer object by calling replace().

  StringBuilder

  StringBuilder is similar to StringBuffer except for one important difference. It is not Synchronized, which means it is not thread-safe. The advantage of StringBuilder is faster performacnace. Incase, s mutable strng will be accessed by multiple threads, and no external synchronization is employed, you must use StringBuffer rather than StringBuilder.

Inheritance

  Inheritance Basics

  Fundamentals

  You can inherit a class by using the extends keyword.
 Although a subclass includes all of the members of its superclass, it cannot access those members of the superclass that have been declared as private. A reference variable of super class can be assigned a reference to any subclass derived from that superclass. Whenever a subclass needs to refer to its immediate superclass, it can do it so by use of the keyword super.
  Super has two general forms first calls the superclass Contructor and the second is used to access a member of the superclass that has been hidden by a member of a subclass. The second use of Super acts somewhat like 'this', except that it always refers to the superclass of the subclass i which it is used. Whether or the not the super() is used the contructor of the super class is created. If the super is not used then the default paramertless contructor of the super class is used.

  Abstract Classes

 There are situations in which you will want to define a superclass that declares the structure of a given abstraction without providing a complete implementation of every method. That is, sometimes you will need to create a superclass that only defines a generalized form that will be shared by all of its subclasses, leaving it to each subclass to fill in the details. Such a class determines the nature of methods that the subclass must implement.
 You can require that certain methods be overridden by subclasses by specifying the abstract type modifier. Thus, a subclass must override these abstract functions.
  To declare a class abstract, you can simply use the keyword abstract in front of the class.

  Final Keyword

 The final keyword has three uses.It can be used to create the equivalent of a named constant. The other two uses of final apply to inheritance. Using final will prevent method overrinding. Methods decalres as final cannot be overridden.
  Sometimes, you want to prevent a class from being inherited. Precede the class declaration with final. Declaring a class as final will implicitly declares all of its method as final too.   It is illegal to declare a class as both final and abstract since an abstract class is incomplete by itself and relies upon its subclasses to provide complete implementations.

Packages and Interface

  Packages

 Packages are containers for classes. They are used to keep the class name space compartmentalized.You can define classes inside packages which are not accessible by code outside that package. you can also declare class members that are exposed only to other members of the same package. This allows your classes to have intimate knowledge of each other, but not expose to that knowledge to the rest of the world.   Packages in Java are simply a mechanism used to organize classes and prevent class name collisions.

  Interfaces

 You can fully abstract a class by using the keyword interface. Using interface you can specify a set of methods that can be implemented by one or more classes. Although they are similar to abstract classes, interfaces have an aditional capability. A class can implement more than one interface.
  Interfaces are syntactically similar to classes but they lack instance variables, and as a general rule, their methods are decalred without any body.

  Nested Interfaces

 An Interface can be decalared a member of a class or another interface. Such an interface is called a member interfac or a nested interface. A nested interface can be declared as public, private or protected. When a nested interface is used outside of its enclosing scope, it must be qualified by the name of the class or interface of which it is a member.
